# Bug 严重度分级标准

## P0 — Critical（阻断性）

必须立即修复，阻断核心流程或存在安全风险。

| 类型 | 示例 |
|------|------|
| 功能不可用 | 核心页面白屏、主流程无法完成 |
| 数据丢失 | 提交成功但数据未保存、误删无确认 |
| 安全漏洞 | XSS 可执行、未认证可访问敏感数据、SQL 注入 |
| 页面崩溃 | JS 未捕获异常导致页面卡死 |
| 数据错乱 | 显示他人数据、ID 错位 |

**判定条件**：满足以下任一
- 用户无法完成核心业务操作
- 存在数据安全风险
- 影响所有用户

## P1 — Major（重要）

严重影响体验，需在当前迭代修复。

| 类型 | 示例 |
|------|------|
| 功能部分失效 | 搜索偶尔不返回结果、分页第二页空白 |
| 严重 UX 问题 | 操作无反馈（点击按钮无响应提示）、加载超 5 秒无 loading |
| 表单问题 | 校验不生效、提交后数据部分丢失 |
| 响应式问题 | 关键内容在常见分辨率下被截断 |
| 性能劣化 | 页面加载 > 5 秒、交互响应 > 2 秒 |
| Network 错误 | 非预期的 4xx/5xx 响应 |

**判定条件**：满足以下任一
- 功能可用但体验显著降低
- 影响部分用户或部分场景
- 有 workaround 但不合理

## P2 — Minor（轻微）

不影响功能，可在后续迭代修复。

| 类型 | 示例 |
|------|------|
| 样式瑕疵 | 对齐偏差、颜色不一致、间距不均 |
| 文案错误 | 错别字、翻译缺失、提示语不准确 |
| 非关键功能 | tooltips 不显示、面包屑路径错误 |
| Console 警告 | 非 error 级别的 warning（如 React key 警告）|
| 可访问性 | 缺少 aria-label、对比度不足 |

**判定条件**：
- 不影响核心功能
- 不影响数据正确性
- 用户可正常使用

## 分级原则

1. **用户视角优先**：从终端用户角度评估影响
2. **宁高勿低**：拿不准时选择更高严重度
3. **安全最高**：任何安全问题至少 P0
4. **组合升级**：多个 P2 集中在同一页面，整体可升级为 P1
