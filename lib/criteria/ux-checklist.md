# UX 质量评分清单

## 评分维度（每项 0-10 分）

### 1. 可用性（Usability）

| 分数 | 标准 |
|------|------|
| 9-10 | 所有核心流程一目了然，无需思考即可完成 |
| 7-8 | 核心流程清晰，偶有犹豫但能自行解决 |
| 5-6 | 部分操作需要摸索，但最终可完成 |
| 3-4 | 多处操作令人困惑，需要帮助文档 |
| 0-2 | 核心流程无法自主完成 |

检查点：
- [ ] 导航结构清晰，当前位置可感知
- [ ] 主要操作入口显眼，不需要寻找
- [ ] 操作反馈及时（成功/失败/加载中）
- [ ] 错误信息具有指导性（说明如何修正）
- [ ] 表单标签清晰，必填标记明确

### 2. 一致性（Consistency）

| 分数 | 标准 |
|------|------|
| 9-10 | 全站风格统一，交互模式一致 |
| 7-8 | 大部分一致，个别页面有偏差 |
| 5-6 | 主要页面一致，次要页面风格混乱 |
| 3-4 | 各页面风格差异明显 |
| 0-2 | 无统一设计语言 |

检查点：
- [ ] 按钮样式统一（主要/次要/危险）
- [ ] 表格/列表样式一致
- [ ] 弹窗/抽屉交互模式统一
- [ ] 颜色使用有语义（成功=绿，错误=红）
- [ ] 间距和字号使用一致

### 3. 响应速度（Performance）

| 分数 | 标准 |
|------|------|
| 9-10 | 所有页面 < 1s 加载，交互即时响应 |
| 7-8 | 大部分页面 < 2s，少数 < 3s |
| 5-6 | 多数页面 2-5s，有 loading 提示 |
| 3-4 | 部分页面 > 5s，且无加载提示 |
| 0-2 | 频繁卡顿、超时、请求失败 |

检查点：
- [ ] 首页加载 < 3 秒
- [ ] 路由切换 < 1 秒
- [ ] 表单提交有即时反馈
- [ ] 长列表有分页或虚拟滚动
- [ ] 无不必要的重复请求

### 4. 无障碍（Accessibility）

| 分数 | 标准 |
|------|------|
| 9-10 | 完全键盘可操作，屏幕阅读器友好 |
| 7-8 | 主要功能键盘可达，有基本 ARIA |
| 5-6 | 部分功能仅鼠标可操作 |
| 3-4 | 大量交互缺少键盘支持 |
| 0-2 | 完全依赖鼠标，无 ARIA 标记 |

检查点：
- [ ] Tab 键可遍历所有交互元素
- [ ] 焦点顺序合理且可见
- [ ] 图片有 alt 属性
- [ ] 颜色不是唯一信息传达方式
- [ ] 对比度满足 WCAG AA（4.5:1）

### 5. 错误处理（Error Handling）

| 分数 | 标准 |
|------|------|
| 9-10 | 所有错误有友好提示且可恢复 |
| 7-8 | 大部分错误有提示，少数需刷新 |
| 5-6 | 有基本错误提示，但信息不够具体 |
| 3-4 | 错误时显示技术信息或白屏 |
| 0-2 | 无错误处理，频繁崩溃 |

检查点：
- [ ] 网络错误有重试选项
- [ ] 404 页面有导航回主页的入口
- [ ] 表单校验显示在对应字段旁
- [ ] 不向用户暴露技术细节（stack trace）
- [ ] 有全局错误边界兜底

## 输出格式

```markdown
## UX 评分

| 维度 | 分数 | 说明 |
|------|------|------|
| 可用性 | 8/10 | 核心流程清晰，设置页入口较深 |
| 一致性 | 7/10 | 主界面统一，弹窗样式有 2 种风格 |
| 响应速度 | 6/10 | 列表页加载 4s，其余正常 |
| 无障碍 | 5/10 | 缺少键盘导航支持 |
| 错误处理 | 7/10 | 有基本提示，部分字段无校验 |
| **综合** | **6.6/10** | |
```
