# Round 2: 边界测试

## 目标

验证输入校验、错误处理、边界条件的健壮性。

## 前提

仅在 Round 1 完成后执行。基于 Round 1 发现的表单和输入点进行针对性测试。

## 测试向量

### 1. 空值测试

对每个表单的每个字段：
- 提交完全空的表单
- 逐个字段留空，其余填有效值
- 验证：应有明确的错误提示，不应 500 错误或白屏

### 2. 超长输入

| 字段类型 | 测试值 |
|---------|--------|
| 文本框 | 重复 `A` × 10000 字符 |
| 数字框 | `99999999999999` |
| 文本域 | 重复段落 × 100 |
| URL 字段 | `http://` + `a` × 2048 |

验证：不应导致页面崩溃、布局溢出、或后端 500

### 3. 特殊字符

```
基础特殊字符：!@#$%^&*()_+-=[]{}|;':",.<>?/`~
Unicode：中文「测试」日文「テスト」emoji「🎉」
HTML 标签：<script>alert(1)</script>
SQL 片段：' OR 1=1 --
换行符：line1\nline2\rline3
制表符：col1\tcol2
```

验证：
- 特殊字符应被正确转义或拒绝
- 不应触发 XSS（检查页面是否弹出 alert）
- 不应触发 SQL 注入（检查是否返回异常数据）

### 4. 数字边界

| 场景 | 测试值 |
|------|--------|
| 负数 | `-1`, `-999` |
| 零 | `0` |
| 小数 | `0.1`, `3.14159` |
| 极大值 | `2147483647`（Int32 最大值）|
| 非数字 | `abc`, `1e99`, `NaN`, `Infinity` |

### 5. 日期边界

| 场景 | 测试值 |
|------|--------|
| 过去日期 | `1970-01-01` |
| 未来日期 | `2099-12-31` |
| 无效日期 | `2024-02-30`, `0000-00-00` |
| 格式错误 | `not-a-date`, `13/32/2024` |

### 6. 文件上传（如有）

- 空文件（0 字节）
- 超大文件（>100MB，仅测试是否有前端限制提示）
- 错误 MIME 类型（将 .txt 改为 .jpg 后缀）
- 特殊文件名：`../../../etc/passwd`、`test file (1).pdf`

## 执行方式

```
对 Round 1 发现的每个表单：
  1. snapshot → 识别所有输入字段及类型
  2. 对每个字段逐一应用上述测试向量
  3. 提交 → 检查响应
  4. 发现问题 → 截图 + 记录 console/network 错误
  5. 清理测试数据（如已创建）
```

## 判定标准

- 有清晰的校验错误提示 → 通过
- 无提示但静默忽略 → P2（UX 问题）
- 页面崩溃 / 白屏 → P0
- 500 错误 → P1
- XSS 弹窗 / 数据泄露 → P0（安全漏洞）
